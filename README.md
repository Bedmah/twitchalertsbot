# TwitchAlertsBot

Telegram-бот для уведомлений о старте Twitch-стримов.
Проект поддерживает подписки на любых стримеров, админ-управление, рассылки и хранение данных в SQLite.

## Что умеет бот

- Подписка на любого Twitch-стримера по логину или ссылке
- Раздел «Рекомендации» с быстрым оформлением подписки
- Раздел «Мои подписки» с отпиской через кнопки
- Разделы «Ссылки» и «Обратная связь»
- Защита от повторных уведомлений при перезапуске стрима (кулдаун)
- Хранение пользователей, подписок и настроек в базе данных

## Меню пользователя

- `Подписаться` — ввод логина или ссылки на канал
- `Рекомендации` — список стримеров для быстрого добавления
- `Мои подписки` — просмотр и отписка через кнопки
- `Ссылки` — полезные ссылки проекта
- `Обратная связь` — контакты для связи

## Админ-панель

- Просмотр пользователей и базовой статистики
- Ограничение доступа к боту для отдельных пользователей
- Настройка списка «Рекомендации» (добавление/удаление)
- Настройка разделов «Ссылки» и «Обратная связь»
- Рассылки: текст, фото с подписью, файл с подписью
- Отчёт по рассылке:
  - сколько успешно отправлено
  - кому не доставлено (chat id + username + причина)
- Настройка лимита подписок:
  - глобально (по умолчанию)
  - персонально для конкретного пользователя

## Антидубли уведомлений о стриме

Если стример быстро перезапускает трансляцию, бот не отправляет повторное уведомление в течение заданного кулдауна (по умолчанию 30 минут).

## Структура проекта

- `bot.py` — основной файл бота
- `app/db.py` — работа с БД и бизнес-данные
- `app/twitch_api.py` — запросы к Twitch API
- `bd/BD/` — активная база данных
- `bd/oldBD/` — архив/миграционные данные
- `run.ps1` — запуск и управление процессом бота в Windows

## Переменные окружения

Используется файл `.env` (в Git не хранится).
Пример заполнения — `.env.example`.

Основные переменные:

- `BOT_TOKEN`
- `TWITCH_CLIENT_ID`
- `TWITCH_CLIENT_SECRET`
- `ADMIN_IDS`
- `STREAM_COOLDOWN_MINUTES`
- `MAX_SUBSCRIPTIONS_DEFAULT`

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python bot.py
```

или через PowerShell-скрипт:

```powershell
powershell -ExecutionPolicy Bypass -File run.ps1 start
```

## Команды `run.ps1`

- `start` — запуск
- `stop` — остановка
- `restart` — перезапуск
- `status` — статус
- `logs -Tail 80` — последние строки логов

## Безопасность

- Секреты и токены не публикуются в репозитории
- Для продакшена рекомендуется регулярно обновлять ключи доступа
